# Telegram-бот для опроса (определение направления работы)

Бот задаёт вопросы по одному (по 4 варианта ответа), считает склонность к разным
**направлениям** и выдаёт пользователю красивую картинку с результатом. Пройти
опрос можно **один раз** на человека. Для суперадминов есть встроенная панель
управления опросом.

## Возможности

**Пользователь:**
- предупреждение, что пройти можно только раз;
- вопросы по одному с inline-кнопками;
- каждый вариант ответа может давать баллы сразу нескольким направлениям;
- картинка-результат (одинаковый дизайн, меняются цифры и описание);
- повторное прохождение блокируется.

**Суперадмин** (кнопка «Админка» видна только ему):
- список всех участников и их результаты;
- полное редактирование опроса: направления, вопросы, варианты ответа и баллы;
- управление суперадминами — добавление/удаление по Telegram ID;
- кнопка «Пересчитать всех» (по умолчанию результаты заморожены и не меняются
  при правках опроса — пересчёт только вручную).

## Стек

Python 3.11+, aiogram 3, SQLAlchemy (async) + SQLite (aiosqlite), Pillow.

## Запуск

1. Установите зависимости:
   ```powershell
   py -3.11 -m venv .venv
   .venv\Scripts\python.exe -m pip install -r requirements.txt
   ```
2. Создайте файл `.env` (можно скопировать из `.env.example`) и заполните:
   ```
   BOT_TOKEN=токен_от_@BotFather
   SUPERADMIN_ID=ваш_Telegram_ID   # узнать у @userinfobot
   DB_PATH=survey.db
   ```
3. Запустите бота:
   ```powershell
   .venv\Scripts\python.exe -m bot.main
   ```

При первом запуске создаётся БД, заливается опрос «Карьерный квест: найди свою
роль» (8 ролей-направлений, 6 вопросов по 8 вариантов) и назначается суперадмин
из `SUPERADMIN_ID`. Опрос можно полностью изменить через админку.

### Как считается результат

Каждый вариант ответа даёт +1 балл своей роли. Побеждает роль, набравшая больше
всего баллов («какая буква встречается чаще»). При равенстве лидеров решающим
считается последний вопрос (№6): берётся роль выбранного там варианта. Логика —
в [bot/scoring.py](bot/scoring.py). Если в админке менять баллы вариантов, можно
сделать так, чтобы ответ давал очки сразу нескольким направлениям.

## Оформление картинки результата (необязательно)

Положите свои файлы в папку `assets/`, чтобы заменить вид картинки:
- `background.png` — фон 1080×1350;
- `font.ttf` / `font-bold.ttf` — шрифты с поддержкой кириллицы.

Без них используется встроенный градиент и системный шрифт.

## Структура

```
bot/
  main.py        — точка входа
  config.py      — чтение .env
  database.py    — подключение к БД
  models.py      — таблицы (направления, вопросы, ответы, баллы, участники, суперадмины)
  seed.py        — демо-опрос и первый суперадмин
  scoring.py     — подсчёт баллов
  image.py       — генерация картинки результата
  keyboards.py   — клавиатуры пользователя
  states.py      — FSM-состояния
  handlers/
    user.py      — прохождение опроса
    admin.py     — админ-панель
assets/          — опциональные фон и шрифты
smoke_test.py    — локальная проверка без запуска бота
```

## Заметки

- Один пользователь = одна запись `Participation`. Сохраняются и «сырые» ответы,
  и снимок результата, поэтому пересчёт возможен, но по умолчанию старые результаты
  не меняются.
- Удаление направления/вопроса не затрагивает уже сохранённые результаты участников.
